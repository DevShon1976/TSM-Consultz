# SECTION 2 — CORE TSM CONCEPTS

## BNCA = Best Next Course of Action

BNCA means:

“What should happen next?”

Examples:

### Construction

* Escalate delayed RFI
* Assign superintendent review
* Flag cost drift

### Healthcare

* Escalate prior authorization
* Route denial to appeals
* Flag AR aging

### FinOps

* Reconcile invoice mismatch
* Assign controller review
* Flag duplicate payment risk

### Legal

* Route contract for review
* Escalate compliance issue
* Assign billing audit

### TaxOps

* Flag missing K-1
* Route IRS notice
* Assign reviewer

### RRD

* Escalate charge-off risk
* Assign collections owner
* Trigger recovery workflow

---

# SECTION 3 — WHAT WIP MEANS

## WIP = Work In Progress

WIP is unfinished operational work.

WIP tells executives:

* what is open
* what is aging
* what is delayed
* what is risky
* who owns it
* what needs action

## WIP Examples

### Healthcare

* prior auth queue
* unresolved denials
* AR aging

### Construction

* RFIs
* submittals
* pay apps
* incident reviews

### FinOps

* unreconciled invoices
* AP exceptions
* close blockers

### TaxOps

* missing documents
* returns awaiting review
* unresolved notices

### Legal

* matter intake backlog
* billing audit exceptions
* contract review queue

### RRD

* recovery queue
* payment disputes
* charge-off exposure

# SECTION 10 — TROUBLESHOOTING

## Problem:

Page not updating.

### Try:

```bash
fly deploy
```

Then hard refresh browser.

---

## Problem:

PM2 app crashed.

### Try:

```bash
pm2 logs
pm2 restart all
```

---

## Problem:

NGINX broken.

### Try:

```bash
sudo nginx -t
sudo systemctl restart nginx
```

---

## Problem:

AI route not responding.

### Check:

```bash
grep -n "/api/bpo/query" server.js
```

---

## Problem:

Fly deploy failing.

### Try:

```bash
fly status
fly deploy
```
