// WIP Intelligence

What is WIP Intelligence?

WIP (Work in Progress) Intelligence identifies the gap between what you've accrued and what's been invoiced — in real time. For FinOps, that means cloud spend that's recorded but not yet matched to a vendor invoice.

📊
Accrued Spend
Cloud costs recorded in your accounting system. What you expect to owe based on usage data.
🧾
Invoiced Amount
What vendors have actually billed you. Should match accruals at period close.
⚠
Variance = The Risk
The gap between accrued and invoiced. Currently $500k — must be resolved before period close or P&L is misstated.
🔴 Current status: $500,000 variance detected. Azure infrastructure invoices unmatched (76% match rate). ACCRUAL_FLUSH action queued. Resolve before period close.

// Accrual-RECON API

The live API behind the charts.

Every KPI card and chart is powered by a real API endpoint. You can call it directly, wire it to your own tools, or trigger it from the Run button in the hub.

Endpoint

POST
/api/wip/finops/accrual-recon

Request body (all fields optional)

{
  "accrued": 4400000,
  "invoiced": 3900000,
  "cloud_provider": "Multi-cloud",
  "period": "Current Month",
  "flush_threshold": 100000
}

Response

{
  "suite": "finops-suite",
  "logic": "ACCRUAL-RECON",
  "status": "VARIANCE_DETECTED",
  "action": "ACCRUAL_FLUSH",
  "message": "$500,000 variance. Manual flush queued.",
  "metrics": { "variance": 500000, "invoice_match_rate_pct": 89 },
  "next_steps": ["Pull cloud billing invoices", "Match to accrual entries", ...]
}
✓ Test it now: The Run Accrual-RECON button in the hub calls this endpoint live. Or open the WIP Dashboard FinOps sector and the data auto-loads.

ACCRUAL_FLUSH is the action returned by the Accrual-RECON API when variance exceeds the flush threshold ($100k by default). It means the variance requires manual GL review before period close. Auto-flush triggers automatically for variances under $100k.
Azure Reserved Instance purchases created unplanned accruals in months 4–6 that don't yet have matching invoices. The fix: pull the Azure billing batch, match to accrual entries, and manually flush the $280k variance before month-end close.
