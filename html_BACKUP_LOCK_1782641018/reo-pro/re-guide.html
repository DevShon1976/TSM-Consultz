WHAT THIS IS
A Deal Intelligence Command Center
The platform connects four specialized apps that each handle a different layer of analysis. You move through them in order — upload a document, run AI nodes to extract intelligence, send that intelligence to a strategist for synthesis, then escalate to an executive portal for board-level reporting. You can also enter any page directly if you already have the context you need.
PAGE 1
RE Document Command
Upload any real estate document. Six specialized AI engines extract findings across acquisition, transaction, mortgage, compliance, and executive dimensions. Output is sent directly into the War Room.
PAGE 2
RE War Room
14 clickable node modules covering every deal dimension. Run individual analyses, use Quick Fire for instant cross-node synthesis, or deploy the Deal Rescue Pack when a transaction is at risk.
PAGE 3
RE Strategist
Receives the War Room session payload and runs 9 strategy modules: risk matrix, compliance stack, pipeline optimization, appraisal playbook, title resolution, REO disposition, market analysis, and executive summary.

Aggregates everything upstream into a board-ready Executive Intelligence Brief with KPI dashboard, deal snapshot, compliance flags, revenue forecast, and 60-day action plan. One-click generation.
⚡ HOW TO START
The fastest path: go to RE Doc Command, drop in any deal document (appraisal, loan app, TRID disclosure, purchase contract), fire all 6 engines, then click SEND TO WAR ROOM →. The War Room will pre-populate with your deal data. From there, run nodes, save outputs, then escalate up the chain.
PLATFORM ARCHITECTURE
The 4-Page Chain Explained
Each page feeds the next via sessionStorage relay. Data flows forward — the War Room receives Doc Command context, the Strategist receives the War Room session, and the Exec Portal aggregates everything. You can enter at any step.
1
RE Doc Command — Upload a PDF, DOCX, XLSX, or paste raw text. Select the document type (appraisal, TRID, loan app, title commitment, etc.). Fire all 6 engines. Each engine reads the same document but through a different analytical lens. When complete, click SEND TO WAR ROOM → — this writes the engine outputs into sessionStorage.
2
RE War Room — Receives the Doc Command payload automatically if you used the Send button. The right panel shows pipeline KPIs. Click any of the 14 node modules to run targeted AI analysis. Save the outputs you want to keep. When ready, click ESCALATE → RE STRATEGIST from the right panel — this writes the full session to localStorage.
3
RE Strategist — Reads the War Room payload from localStorage. Shows which modules were run and outputs received. Click any of the 9 strategy modules to generate deeper analysis. Use BUILD FULL BRIEF for a compiled document. Then ESCALATE → EXEC PORTAL to push everything forward.
4
RE Exec Portal — Receives the full chain payload. Click GENERATE BRIEF for the AI-powered executive intelligence brief. The KPI dashboard auto-populates from upstream data. Use DEAL RESCUE if critical issues were flagged. Export or present directly from this page.
⚠ IMPORTANT — SESSION STORAGE
Data passes between pages via browser sessionStorage and localStorage. Do not close all browser tabs between steps — this will clear the session. If you need to pause, export your output from the right panel first using the EXPORT (.TXT) button before closing. You can always re-enter the chain at any step manually.

APP GUIDE · WAR ROOM FEATURE
Deal Rescue Pack — Transaction Emergency System
The Deal Rescue Pack is the War Room's most powerful single feature. It's a structured intake form designed specifically for transactions that are in distress — where earnest money is at risk, closing is delayed, or something has gone wrong. You fill in the deal, select what's broken, and the AI generates a complete remediation plan with a health score, ranked issue list, and specific action steps for every party involved.
When to Use the Deal Rescue Pack
Use the Rescue Pack whenever a transaction has a problem that threatens closing. It's not for routine analysis — it's for crisis response. If you're just exploring a deal, use the War Room nodes instead. Rescue Pack is for when the deal is actively at risk.
Appraisal came in low Buyer financing denied or at risk Title defect discovered Inspection killed the deal Underwriting conditions blocking CTC Closing date at risk of missing
STEP-BY-STEP
1
DEAL IDENTIFICATION — Fill in the 6 fields at the top: Property Address, Contract Price, Loan Type, Contract Status, Closing Date, and Days Until Closing. All fields accept free-form text — use real data. The more specific, the better the output. Even 123 Desert Ridge Dr, Phoenix AZ is more useful than leaving it blank.
2
SELECT RESCUE SCENARIOS — Click every scenario that applies to your deal. You can select multiple. Each selected scenario adds a specialized remediation module to the analysis. Don't pre-select — read each label and choose only what's actually broken.
3
ADDITIONAL DETAILS — Type a plain-English description of the specific problem. Include: what happened, who's involved, when it was discovered, any amounts discussed. Example: Appraiser used bad comps. Seller won't budge on price. Buyer can't increase down payment. HOA super-lien of $4,200 also found on title search today.
4
DOLLAR AMOUNTS AT RISK — Enter Earnest Money and the Appraisal Gap / Issue Amount if known. These figures appear in the health score calculation and make the remediation plan dollar-specific. A blank earnest money field means the AI can't calculate loss exposure accurately.
5
RUN TRANSACTION RESCUE PACK — Click the fire button. The AI runs a single comprehensive analysis (not multiple calls) covering all selected scenarios simultaneously. The output includes: a Transaction Health Score (0–100), Overall Risk Level (LOW / MEDIUM / HIGH / CRITICAL), an Issues Found section with dollar impact per issue, a Remediation Plan for each scenario with numbered actions and party responsibilities, a Priority Order (24hr / 72hr / this week), and a Projected Closing Delay estimate.

⚡ PRO TIP — READING THE HEALTH SCORE
The Transaction Health Score (0–100) is generated from the AI's reading of severity, complexity, and timeline pressure — not a formula. 75–100 = manageable issues, deal likely closes. 50–74 = medium risk, active intervention required. 25–49 = high risk, deal in danger. 0–24 = critical, earnest money at immediate risk. Use the score as a communication tool with clients and teams — not just as internal data.
⚡ PRO TIP — MULTI-SCENARIO POWER
Selecting multiple scenarios doesn't just run them in parallel — it surfaces cross-scenario interactions that a single-scenario analysis misses. A deal with both an appraisal gap AND a title defect AND UW conditions creates a compounding problem that the AI will model holistically. Always select every scenario that applies, even if you think one is minor.
⚠ SAVE THE RESCUE OUTPUT
After the Rescue Pack completes, the output is automatically saved to the session. But also copy the health score and key action items into your own notes or CRM immediately. The Rescue Pack output is the single most shareable artifact from the platform — it can be copy-pasted into a client email, shared with the listing agent, or used as a remediation brief in an internal meeting.

⚡ HOW THE RECOMMENDED PATH WORKS
After every node run, the system scans the AI output for specific keywords (lien, TRID, reconcile, fall-through, etc.) and scores all 7 tools by how many matching keywords appear. The BEST MATCH tile is the highest-scoring tool. Steps 2–6 are the next-highest in ranked order. This means the path is built from the actual content of your node's output — not a generic recommendation. A compliance-heavy output surfaces the Compliance Shield first. A cash flow issue surfaces the Financial Analyzer first.

ADVANCED GUIDE
Getting Better Outputs — Pro Techniques
The platform's AI is only as sharp as the context it receives. These techniques consistently produce more specific, more actionable, and more accurate outputs from every node, rescue pack, and engine.
1. Specificity = Quality
The nodes run with default deal parameters (22 loans, $18.4M, Phoenix metro). Every real number you add — actual property address, real loan amount, actual days until closing — dramatically improves output quality. Use the TSM RESCUE widget (bottom of screen) to inject your specifics: "I have a $412K FHA loan with 14 days to close. Appraisal came in at $388K. Seller is offering $5K concession. Buyer has $12K in reserves." Then run the relevant node and it will use your numbers.
2. Stack Nodes for Cross-Reference
Nodes share session context through the saved output list. Run Phoenix Market Analysis first to establish comps and market conditions. Then run Appraisal Gap Rescue — the AI will reference the market context established in the first run. This stacking effect produces outputs that are internally consistent and reference each other, making the final synthesis much stronger.
3. Use the Feed Output as Input
After a node completes, copy a key section from the feed and paste it into the Rescue Widget as context for your next question. Example: Phoenix Market Analysis shows median DOM is 47 days → paste that finding → ask the Rescue Widget: "Given 47-day DOM, what's the carrying cost risk if this deal falls and relists?" You're chaining the AI's own output back to it for deeper analysis.
4. Name Every Report Intentionally
In the Synthesis Report title field, use a format like [Address] · [Issue Type] · [Date]. Example: 123 Desert Ridge · Appraisal Gap + Title Lien · June 7 2026. This makes the escalated payload readable by the Strategist and Exec Portal as named context — the AI will reference this title in its outputs, making the brief more professional and specific.
5. Document Types → Engine Accuracy
In RE Doc Command, selecting the correct document type is the single biggest lever for engine output quality. The engines use the document type to calibrate their extraction rules. A TRID disclosure classified as "Other" will produce generic output. The same file classified as "TRID Disclosure" will produce a violation-by-violation breakdown with cure amounts.
6. The Rescue Widget — Live Q&A
The TSM RESCUE · LIVE DEAL SUPPORT widget at the bottom of the War Room is a freeform AI chat that runs in the same context as the session. Use it to: ask follow-up questions on any node output, get a quick sanity check on a number, draft a paragraph for a client email, or probe a scenario the nodes didn't cover. It's not saved to the session automatically — copy anything useful into the synthesis textarea.

ADVANCED GUIDE
Full Workflow Playbook — Scenario by Scenario
Real-world workflows for the most common use cases. Each playbook shows exactly which apps to open, which nodes to run, what to save, and what the final output looks like.
PLAYBOOK A — DEAL IN DISTRESS
Transaction is at risk. Closing in <14 days. Multiple issues.
1
War Room → Deal Rescue Pack tab — Fill all fields with real deal data. Select every applicable scenario. Include EM and gap amounts. Fire.
2
Read the Health Score — If CRITICAL, go to NODE VIEWER tab immediately and open the most relevant remediation tool while the rescue analysis is still in the feed.
3
Run Closing Delay Rescue node (REAX) — Gets the daily carrying cost, revised timeline, and escalation memo.
4
Run RESPA/TRID Audit node (REAC) if any compliance issues were flagged — Gets cure amounts and CD delivery requirements.
5
Synthesize → Escalate → Exec Portal → Generate Brief — Share the brief with all parties involved. The brief contains the health score, all issues, all remediation steps, and a ranked priority order.
PLAYBOOK B — NEW LOAN FILE REVIEW
Brand new file just came in. Need a full picture before assigning a processor.
1
RE Doc Command — Upload the loan application (1003). Set document type to "Loan Application." Fire all 6 engines. Send to War Room.
2
War Room → Pre-Qualification Engine (REAF) — Gets DTI, max loan amount, and product recommendation.
3
War Room → Underwriting Conditions (REAM) — Gets a predicted conditions list and processor work queue.
4
War Room → RESPA/TRID Audit (REAC) — Pre-flight compliance check before the LE is issued.
5
Save all 3 outputs → Synthesize → Export as .TXT — Use as the file setup document for the processor.
PLAYBOOK C — WEEKLY PORTFOLIO REVIEW
Monday morning. Need a full pipeline read and priority list for the week.
1
War Room → Quick Fire: Highest Risk Deal — Gets the single most at-risk file and what to do in the next 48 hours.
2
Quick Fire: Compliance Sweep — Gets all compliance items ranked by dollar liability. Assign owners before the standup.
3
Pipeline Velocity node (REAM) — Gets the pull-through gap dollar impact and the top 5 fall-through risk loans.
4
Quick Fire: Market Update — Quick Phoenix market context for any client conversations this week.
5
Synthesize → Escalate → Exec Portal → Board Report — Weekly board brief with KPIs, risks, and actions. Export and distribute.
PLAYBOOK D — REO ASSET DECISION
Portfolio has an REO asset. Need to decide: retail list, bulk sale, or auction.
1
RE Doc Command — Upload the BPO or disposition report. Document type: "REO Disposition." Fire engines.
2
War Room → BPO/Valuation Engine (REAI) — Gets the adjusted value, comp methodology, and documentation.
3
War Room → Phoenix Market Analysis (REAI) — Gets submarket conditions and absorption rate for the asset's location.
4
Strategist → REO Disposition Strategy module — Retail vs. bulk vs. auction analysis with net proceeds projection per path.
5
Exec Portal → REO 60-Day Plan — Timeline and action plan for the selected disposition path. Share with the asset manager.
